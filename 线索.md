# 《回声工牌》线索与结局触发路线 v4

> 用途：制作和实现参考。本文只规定玩家如何推进、设置哪些状态、怎样分流结局。
> 本版结构：顾路工作台 -> 头像退出登录 -> 工牌登录 -> 工牌复核视图 -> 五种分流。
> TMP 仍是独立彩蛋，不参与顾路事件正式结局。

## 2. 推进路线

### 2.1 首页新闻 -> 旧口令登录

玩家看到：

- 首页报道标题为“一名24岁男子从南洄科技园C座坠落”。
- 报道副标题为“警方排除刑事案件”。
- 家属口述暗示旧口令与生日 `0606` 有关。

玩家动作：

- 点击“开始调查”。
- 在 `login.html` 输入 `0606`。

触发：

```js
flags.loggedInWithOldPassword = true;
```

进入：

- `workbench.html`
- 默认显示顾路工作台，右上角头像为顾路账号。

---

### 2.2 顾路工作台调查

玩家看到：

- 右上角头像显示顾路账号。
- 消息礼貌。
- 文档整洁。
- 待办闭环。
- DING 普通。
- AI 听记摘要无风险。
- 人事、领导、同事都配合调查。

建议触发：

```js
evidence.guWorkbenchChecked = true;
evidence.surfaceChatsFriendly = true;
evidence.surfaceDocsClean = true;
evidence.surfaceTodoClosed = true;
evidence.surfaceAiSummaryClean = true;
```

顾路账号可疑点：

- 当前账号不是玩家账号，而是顾路账号。
- 头像账号信息显示 `GL-0606 / 凭证待回收`。
- 设备归还审批没有附件。
- AI 听记摘要时长对不上原音频时长。
- 头像菜单里有“退出登录”。

如果玩家此时归档：

```js
flags.signedOffTooEarly = true;
ending = "careless-fool";
```

---

### 2.3 头像退出登录 -> 工牌登录

入口来源：

- 顾路工作台右上角头像。
- 头像菜单里的“退出登录”。
- 账号信息里的 `GL-0606 / 凭证待回收`。

玩家动作：

- 点击顾路头像。
- 选择“退出登录”。
- 进入新的“南洄凭证复核登录”界面。
- 上传或拖入顾路工牌 HTML。

顾路工牌校验字段：

- 姓名：顾□□□
- ID：`GL-0606`
- 公司：南洄科技有限公司
- 类型：`Recovered Staff Credential`

触发：

```js
evidence.guAvatarMenuOpened = true;
evidence.loggedOutFromGuAccount = true;
evidence.badgeLoginPageOpened = true;
evidence.glBadgeVerified = true;
evidence.badgeRecoveryViewUnlocked = true;
```

如果上传的是 Base64：

```js
flags.guBadgeUploadedAsBase64 = true;
```

进入：

- 工牌复核视图。

### 2.4 工牌复核视图调查

工牌复核视图与顾路工作台模块一致，但数据不同。

消息页：

- 顾路 6 月 5 日仍正常工作。
- 郑明真实语气尖酸刻薄。
- 系统机器人出现 `route-bot`。

触发：

```js

```

文档页：

- 归档版写“补交设备”。
- 修订前写“补交证据封包”。
- 修订时间晚于坠落时间。

触发：

```js
evidence.docEditedAfterDeath = true;
```

日历页：

- 14:30 日程创建者是 `route-bot@nhcloud.local`。

触发：

```js
evidence.calendarCreatedByBot = true;
```

待办页：

- “补交设备”由门禁联动回执自动完成。

触发：

```js
evidence.todoAutoCompleted = true;
```

DING 页：

- 技术信息出现 `zhao.yan/security` 与 trace id。

触发：

```js
evidence.dingSignatureZhao = true;
```

AI 表格：

- 筛选 `GL-0606` 后出现 `C-OLD-0606`。
- hidden columns 显示 `intervention=L3`。

触发：

```js
evidence.aiTableHiddenRoute = true;
```

AI 听记 / 会议：

- 摘要缺失关键句。
- 原音频比摘要长 4 分 06 秒。
- 输入 `0406` 恢复尾段。

触发：

```js

```

- 旧记录显示“补交证据封包”。
- 新记录显示“补交设备”。

```js

```

顾路工作台/复核视图对照：

- 玩家同时导出顾路工作台归档记录和工牌复核视图记录。
- 两份记录同 ID、同时间、不同字段。

触发：

```js

```

离线封包：

- 在隐藏真实路线中，玩家通过顾路工牌回声找到最后一次离线保存。

触发：

```js
evidence.offlinePackageRecovered = true;
```

## 4. 结局判断

### 4.1 最坏结局：草率的笨蛋

```js
if (
  evidence.guWorkbenchChecked &&
  evidence.surfaceChatsFriendly &&
  evidence.surfaceDocsClean &&
  evidence.surfaceTodoClosed &&
  !evidence.badgeLoginPageOpened &&
  !evidence.badgeRecoveryViewUnlocked &&
  flags.signedOffTooEarly
) ending = "careless-fool";
```

### 4.2 普通结局：错误的人

```js
if (
  evidence.badgeRecoveryViewUnlocked &&
  flags.bloodMessageUnlocked &&
  flags.misledByZheng &&
  !evidence.dingSignatureZhao &&
  !evidence.meetingLast0406Recovered &&
  !evidence.visitorRecordEvidencePackage
) ending = "normal-wrong-person";
```

### 4.3 彩蛋结局：工牌回声

```js
if (
  evidence.glBadgeVerified &&
  flags.guBadgeUploadedAsBase64 &&
  flags.echoPhraseFound &&
  flags.echoRouteOpened
) ending = "badge-echo";
```

### 4.4 真实结局：系统不能替人作证

```js
const trueEndingEvidence = [
  evidence.badgeRecoveryViewUnlocked,
  evidence.normalWorkBeforeDeath,
  evidence.docEditedAfterDeath,
  evidence.calendarCreatedByBot,
  evidence.todoAutoCompleted,
  evidence.dingSignatureZhao,
  evidence.aiTableHiddenRoute,
  evidence.aiNoteMissingSentence,
  evidence.meetingLast0406Recovered,
  evidence.visitorRecordEvidencePackage,
  evidence.guAndRecoveryMismatchProved
].every(Boolean);

if (
  trueEndingEvidence &&
  flags.submittedToPolice &&
  flags.submittedToThirdParty
) ending = "true-ending";
```

### 4.5 隐藏真实结局：第二层也会说谎

```js
if (
  trueEndingEvidence &&
  evidence.originalMessageHash &&
  evidence.falseBloodMessageResolved &&
  evidence.offlinePackageRecovered &&
  flags.guBadgeUploadedAsBase64 &&
  flags.echoPhraseFound &&
  flags.submittedToPolice &&
  flags.submittedToMedia &&
  flags.submittedToThirdParty
) ending = "hidden-true-ending";
```

## 5. TMP 独立彩蛋

触发：

```js
if (uploadedBadge.type === "TMP") {
  flags.tmpEasterEgg = true;
}
```

后续：

- 在工牌登录界面上传 TMP 工牌。
- 系统提示“该凭证不属于顾路回收池”。
- 输入 TMP 工牌入场日期。

结局：

```js
if (flags.tmpEasterEgg && tmpDateMatched) {
  flags.tmpWorkerEndingSeen = true;
  ending = "tmp-observer-number";
}
```

限制：

- 不设置主线 evidence。
- 不影响 `careless-fool`、`normal-wrong-person`、`badge-echo`、`true-ending`、`hidden-true-ending`。
- 不证明顾路是临时工。

---

## 6. 实现顺序

1. 工作台默认显示顾路账号正常版。
2. 右上角头像显示顾路身份。
3. 头像菜单加入“退出登录”，退出后进入工牌登录页。
4. 顾路工牌登录成功后设置 `badgeRecoveryViewUnlocked`。
5. 消息、文档、日历、待办、DING、AI 表格、AI 听记支持顾路工作台/工牌复核视图两套数据。
6. 加入血迹催命文误导与原始哈希纠正。
7. 归档页实现最坏、普通、彩蛋、真实、隐藏真实结局。
8. 最后加入 TMP 独立彩蛋。

---

## 7. 结局优先级

1. `hidden-true-ending`
2. `badge-echo`
3. `true-ending`
4. `normal-wrong-person`
5. `careless-fool`
6. TMP 独立彩蛋单独记录，不覆盖主线结局。
